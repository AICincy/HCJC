# JCStream Project Notes

## Overview
JCStream is a real-time inmate tracking system for Hamilton County (Ohio) Justice Center. It tracks bookings, charges, and releases across multiple data sources (HCSO roster, court cases, dispatch records, etc.).

## Key Date Reference
May 14, 2026 (Thursday) is used as the standard reference date in tests and documentation.

## Project Structure

### `/scraper/`
Data collection and parsing:
- `client.py` - HTTP client with session management
- `models.py` - Pydantic models for Inmate, Charge, ChangeEvent, etc.
- `orc.py` - Ohio Revised Code offense lookup and degree classification
- `sweep.py` - Main roster fetch from HCSO public API
- `sweep_guards.py` - Guards/continuity checks for sweep consistency
- `parsers.py` - HTML parsing for HCSO detail pages and court records
- `cfs.py` / `cfs_pdi.py` - Police dispatch (Computer-Aided Dispatch) feeds
- `shootings.py` - Officer-involved shooting incidents
- `store.py` - Local JSON storage for current roster and changelog
- `match.py` - Fuzzy matching between dispatches and inmates
- `photos.py` - Inmate mugshot downloading and processing
- `courtclerk.py` - Hamilton County Common Pleas Court case lookup
- `pra.py` / `pra_capias.py` / `pra_base.py` - Public Records Act requests (capias warrants, etc.)
- `ingest_issue.py` - GitHub issue templating for data corrections

### `/web/`
Web rendering and templates:
- `build.py` - Main site builder (reads JSON, renders Jinja templates)
- `shape.py` - View-model helpers (per-inmate / per-snapshot helpers)
- `classify.py` - Pure helpers: data parsing, classification, tier/race/sex expansion
- `/templates/` - Jinja2 templates for HTML + XML (feed.xml)
- `/static/` - CSS, JavaScript, vendor libraries

### `/data/`
Runtime data:
- `current.json` - Current roster snapshot (generated by sweep)
- `changelog.json` - Historical events (booked, released, updated)
- `orc_offenses.json` - ORC code lookup table (title + degree)
- `orc_caselaw.json` - Case law index by ORC code
- `explainers.json` - Contextual explanations for charges
- `surnames.txt` - Common surnames for name-based filtering (sweep guards)
- `photos/` - Downloaded mugshots

### `/tests/`
Regression test coverage for:
- Parsing (HCSO detail pages)
- Models and validation
- ORC lookups and offense classification
- Shape/view-model helpers
- Build rendering
- Data matching and deduplication

### `/docs/`
Generated site output (gitignored during build, committed to Pages)

## Architecture Decisions

### Circular dependency resolution
- `build.py` imports from `classify.py` and `shape.py` but only re-exports them
- `shape.py` imports from `classify.py` (no re-import)
- `classify.py` has no web imports (pure functions)

### Date handling
- All inmate.booking_date values are MM/DD/YY format (2-digit years)
- 2-digit year rule: 70+ → 19XX, <70 → 20XX (supports 1970–2069 range)
- Sentinel dates like "1/1/70" (epoch) are parsed as valid but filtered downstream
- ISO 8601 (UTC) used internally for events (timestamp_utc)

### Tier classification (felony vs. misdemeanor)
1. Try explicit degree suffix in charge description (e.g., "ASSAULT F4" → F4)
2. Fall back to ORC lookup in orc_offenses.json
3. Fall back to case venue: Common Pleas → felony, Municipal → misdemeanor
4. Return None if no signal

### Offense categorization
- Primary offense chapter determined from most-serious charge
- Ranking: violence > sex > robbery > theft > drugs > weapons > obstruction > etc.
- CSS color classes tied to chapter (2903 = red, 2925 = orange, etc.)

### Sweep guards
- Deduplication: booking_number is YYNNNNNN (6-digit sequence per year)
- New inmates must satisfy basic consistency checks before entry
- Continuity: released inmate can re-appear on new booking, but not within 24h

## Configuration

### Environment Variables (optional)
- `JCSTREAM_SITE_BASE_URL` - URL path prefix (e.g., `/jcstream`), default: derived from GitHub repo
- `JCSTREAM_SITE_URL` - Absolute origin (e.g., `https://example.com`), used for feeds
- `JCSTREAM_CNAME` - Custom domain (if set, site_url = `https://<cname>`)
- `JCSTREAM_BASE_URL` - HCSO HTTP origin (rarely override), default: `https://www.hcso.org`
- `JCSTREAM_HCSO_BEARER_TOKEN` - Auth token for HCSO API (if required)

### Data paths (relative to repo root)
- `data/current.json` - Current roster (source)
- `data/changelog.json` - Event history (source)
- `data/orc_offenses.json` - ORC lookup (source, from codes.ohio.gov)
- `data/orc_caselaw.json` - Case law index (optional, for /statute/ page)
- `data/explainers.json` - Charge explanations (optional)
- `data/surnames.txt` - Surname list (used by sweep guards)
- `web/_dist/` - Build output (generated, gitignored)
- `docs/` - GitHub Pages output (committed, serves the public site)

## Testing

Run all tests:
```bash
python -m pytest tests/
```

Run specific test file:
```bash
python -m pytest tests/test_build.py -v
```

Run with coverage:
```bash
python -m pytest tests/ --cov=scraper --cov=web
```

## Building the Site

```bash
python -m web.build /path/to/output/dir
# or
python -m web.build  # defaults to docs/
```
